# 产品管理测试用例

## 模块说明
- **模块名称**: 产品管理 (Product Management)
- **接口基础路径**: `/products`
- **测试目标**: 验证产品增删改查功能的正确性
- **认证要求**: 所有接口需要有效的Token

---

## 1. 创建产品

### 接口信息
| 项目 | 内容 |
|------|------|
| 接口名称 | 创建产品 |
| 请求方法 | POST |
| 接口路径 | /products |
| 接口说明 | 创建新产品信息 |

### TC-PROD-001: 正常创建产品
| 用例属性 | 内容 |
|----------|------|
| 用例编号 | TC-PROD-001 |
| 用例名称 | 正常创建产品 |
| 优先级 | P0 |
| 测试类型 | 正向测试 |

**前置条件:**
1. 用户已登录并获取有效Token

**测试步骤:**
1. 发送 POST 请求到 `/products`
2. 请求头携带有效Token
3. 请求体包含完整的产品信息

**请求参数:**
```json
{
  "name": "测试产品A",
  "price": 199.99,
  "description": "这是一个测试产品",
  "stock": 100
}
```

**预期结果:**
| 检查项 | 预期值 |
|--------|--------|
| HTTP状态码 | 200 |
| 响应code | 200 |
| 响应message | "创建成功" |
| 返回data包含 | id, name, price, description, stock, created_at, updated_at |
| 数据库验证 | 产品数据已写入products表 |

## 接口依赖关系
```
登录获取Token
    ↓
产品管理接口（需要Token）
    ├── POST /products - 创建产品
    ├── GET /products - 获取列表
    ├── GET /products?id=x - 获取详情
    ├── PUT /products/{id} - 更新产品
    └── DELETE /products/{id} - 删除产品
```
