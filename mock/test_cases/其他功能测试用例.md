# 其他功能测试用例

## 模块说明
- **模块名称**: 其他功能 (Other Functions)
- **包含接口**: 数据提交、文件上传、健康检查、服务器信息
- **测试目标**: 验证系统辅助功能的正确性
- **认证要求**: 除初始化接口外，其他需要有效的Token

---

## 1. 数据提交接口

### 接口信息
| 项目 | 内容 |
|------|------|
| 接口名称 | 提交数据 |
| 请求方法 | POST |
| 接口路径 | /api/data |
| 接口说明 | 提交name和value数据，支持query和body参数 |

### TC-OTHER-001: 正常提交数据-body参数
| 用例属性 | 内容 |
|----------|------|
| 用例编号 | TC-OTHER-001 |
| 用例名称 | 正常提交数据-body参数 |
| 优先级 | P1 |
| 测试类型 | 正向测试 |

**前置条件:**
1. 用户已登录并获取有效Token

**测试步骤:**
1. 发送 POST 请求到 `/api/data`
2. 使用JSON body传递参数

**请求参数:**
```json
{
  "name": "test_data",
  "value": "123"
}
```

**预期结果:**
| 检查项 | 预期值 |
|--------|--------|
| HTTP状态码 | 200 |
| 响应code | 200 |
| 响应message | "数据提交成功" |
| 返回data包含 | name, value, timestamp |
| value类型 | 整数类型 |

## 接口汇总

| 接口路径 | 方法 | 需要Token | 说明 |
|----------|------|-----------|------|
