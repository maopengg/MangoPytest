# 认证模块测试用例

## 模块说明
- **模块名称**: 认证模块 (Auth Module)
- **接口基础路径**: `/auth`
- **测试目标**: 验证用户登录和注册功能的正确性

---

## 1. 用户登录接口

### 接口信息
| 项目 | 内容 |
|------|------|
| 接口名称 | 用户登录 |
| 请求方法 | POST |
| 接口路径 | /auth/login |
| 接口说明 | 用户使用用户名和密码登录系统，获取访问令牌 |

### TC-AUTH-001: 正常登录成功
| 用例属性 | 内容 |
|----------|------|
| 用例编号 | TC-AUTH-001 |
| 用例名称 | 正常登录成功 |
| 优先级 | P0 |
| 测试类型 | 正向测试 |

**前置条件:**
1. 系统中已存在用户 `testuser`
2. 用户密码为 MD5 加密后的 `482c811da5d5b4bc6d497ffa98491e38`

**测试步骤:**
1. 发送 POST 请求到 `/auth/login`
2. 请求体包含正确的用户名和密码

**请求参数:**
```json
{
  "username": "testuser",
  "password": "482c811da5d5b4bc6d497ffa98491e38"
}
```

**预期结果:**
| 检查项 | 预期值 |
|--------|--------|
| HTTP状态码 | 200 |
| 响应code | 200 |
| 响应message | "登录成功" |
| 返回data包含 | user_id, username, role, token |
| token格式 | 以"mock_token_"开头 |
