# 审批流程测试用例

## 模块说明
- **模块名称**: 审批流程 (Approval Workflow)
- **接口基础路径**: `/reimbursements`, `/dept-approvals`, `/finance-approvals`, `/ceo-approvals`, `/workflow`
- **测试目标**: 验证四级审批流程（D→C→B→A）的正确性
- **认证要求**: 所有接口需要有效的Token
- **审批流程**: 
  ```
  D级: 报销申请 → C级: 部门审批 → B级: 财务审批 → A级: 总经理审批
  ```

---

### TC-APPROVAL-001: 正常创建报销申请
| 用例属性 | 内容 |
|----------|------|

## 审批状态流转图

```
                    ┌─────────────────┐
                    │     pending     │
                    │   (待审批)      │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
              ▼              │              ▼
    ┌─────────────────┐     │    ┌─────────────────┐
```
