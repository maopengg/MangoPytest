# 用户管理测试用例

## 模块说明
- **模块名称**: 用户管理 (User Management)
- **接口基础路径**: `/users`
- **测试目标**: 验证用户增删改查功能的正确性
- **认证要求**: 所有接口需要有效的Token

---

## 1. 获取用户列表

### 接口信息
| 项目 | 内容 |
|------|------|
| 接口名称 | 获取用户列表 |
| 请求方法 | GET |
| 接口路径 | /users |
| 接口说明 | 获取所有活跃用户列表（不包含已删除用户） |

### TC-USER-001: 正常获取用户列表
| 用例属性 | 内容 |
|----------|------|
| 用例编号 | TC-USER-001 |
| 用例名称 | 正常获取用户列表 |
| 优先级 | P0 |
| 测试类型 | 正向测试 |

**前置条件:**
1. 用户已登录并获取有效Token
2. 系统中存在至少一个活跃用户

**测试步骤:**
1. 发送 GET 请求到 `/users`
2. 请求头携带有效Token

**请求头:**
```
X-Token: mock_token_xxx
```

**预期结果:**
| 检查项 | 预期值 |
|--------|--------|
| HTTP状态码 | 200 |
| 响应code | 200 |
| 响应message | "获取成功" |
| 返回data类型 | 数组列表 |
| 列表项包含 | id, username, email, full_name, role, status, created_at, updated_at |
| 密码字段 | 不包含password字段 |
| 用户状态 | 所有返回用户status为"active" |

---

### TC-USER-002: 未授权访问用户列表
| 用例属性 | 内容 |
|----------|------|
| 用例编号 | TC-USER-002 |
| 用例名称 | 未授权访问用户列表 |

## 接口依赖关系
```
登录获取Token
    ↓
用户管理接口（需要Token）
    ├── GET /users - 获取列表
    ├── GET /users?id=x - 获取详情
    ├── PUT /users/{id} - 更新用户
    └── DELETE /users/{id} - 删除用户
```
