# 订单管理测试用例

## 模块说明
- **模块名称**: 订单管理 (Order Management)
- **接口基础路径**: `/orders`
- **测试目标**: 验证订单创建、查询、更新、删除功能的正确性
- **认证要求**: 所有接口需要有效的Token
- **业务规则**: 
  - 创建订单时会自动扣减产品库存
  - 订单金额 = 产品单价 × 数量
  - 库存不足时无法创建订单

---

## 1. 创建订单

### 接口信息
| 项目 | 内容 |
|------|------|
| 接口名称 | 创建订单 |
| 请求方法 | POST |
| 接口路径 | /orders |
| 接口说明 | 创建新订单，自动扣减库存并计算金额 |

### TC-ORDER-001: 正常创建订单
| 用例属性 | 内容 |
|----------|------|
| 用例编号 | TC-ORDER-001 |
| 用例名称 | 正常创建订单 |
| 优先级 | P0 |
| 测试类型 | 正向测试 |

**前置条件:**
1. 用户已登录并获取有效Token
2. 系统中存在ID为1的产品，库存 >= 10
3. 系统中存在ID为1的用户

**测试步骤:**
1. 记录产品ID=1的当前库存
2. 发送 POST 请求到 `/orders`
3. 请求体包含订单信息

**请求参数:**
```json
{
  "product_id": 1,
  "quantity": 5,
  "user_id": 1
}
```

**预期结果:**
| 检查项 | 预期值 |
|--------|--------|
| HTTP状态码 | 200 |
| 响应code | 200 |
| 响应message | "创建成功" |
| 返回data包含 | id, order_no, product_id, quantity, user_id, unit_price, total_amount, status |
| status | "pending" |
| total_amount | unit_price × quantity |
| 库存验证 | 产品ID=1的库存减少5个 |

## 业务流程测试

### TC-ORDER-017: 完整订单业务流程
| 用例属性 | 内容 |
|----------|------|
| 用例编号 | TC-ORDER-017 |
| 用例名称 | 完整订单业务流程 |
| 优先级 | P1 |
| 测试类型 | 集成测试 |

**业务流程:**
```
1. 查询产品库存
    ↓
2. 创建订单（库存扣减）
    ↓
3. 查询订单详情（验证数据）
    ↓
4. 更新订单状态为"paid"
    ↓
5. 更新订单状态为"shipped"
    ↓
6. 更新订单状态为"completed"
    ↓
7. 查询订单列表（验证状态）
```

**验证点:**
- 每个步骤的响应符合预期
- 库存变化正确
- 订单状态流转正确
- 金额计算正确

## 接口依赖关系
```
登录获取Token
    ↓
订单管理接口（需要Token）
    ├── POST /orders - 创建订单（依赖产品、用户数据）
    ├── GET /orders - 获取列表
    ├── GET /orders/{id} - 获取详情
    ├── PUT /orders/{id} - 更新订单
    └── DELETE /orders/{id} - 删除订单
```
